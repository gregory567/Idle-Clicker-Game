Projektidee:
"Im Rahmen des IT-Projekts haben wir ein Browser Game entwickelt, bei dem Pizzen erstellt werden und das Ziel ist, am Ende die meisten Pizzen erstellt zu haben.
Anfangs gibt es einen einzigen Button zum Aufwärmen von Pizzen, welche automatisch verkauft werden.
Im weiteren Spielfortschritt schaltet der User durch jedes Level neue Features frei: zum Beispiel eine Pizza-Maschine zur Produktion oder einen automatischen Pizza-Einkäufer."

Implementierung:
"Die Funktionalität unseres Klicker-Games haben wir rein in javascript programmiert und zur Darstellung wurde neben HMTL auch mit CSS gearbeitet.
Wir haben uns am Anfang des Projekts für Visual Studio Code als Entwicklungsumgebungs und für Github als Code-Repository entschieden. Während VS Code für alle Projektmitglieder bekannt war, mussten wir den Umgang mit Branches in Github erstmals umsetzen und haben pro Sprint jeweils eine Branch erstellt, damit strukturierte und nachvollziehbare Entwicklung möglich ist. Eine Unterteilung der Branches nach neuen Funtkionalitäten war nicht notwendig, weil alle Features zusammenhängend sind und konkrete Abgrenzungen nur selten möglich sind."

Projekt-Planung:
"Die Projektorganisation und -Entwicklung wurde nach Kanban umgesetzt. Mithilfe von dem Planungstool Trello visualisieren wir unser Kanban-Board: Eigene Spalten zum Backlog, in Bearbeitung oder fertige Sprints sind als Spalten zu sehen. Swimmlanes sind für unsere Projektgröße nicht notwendig und wurden weggelassen.
Die zweiwöchigen Sprints beginnen mit der Erstellung der Karten für den kommenden Sprint und enden stets mit einem internen Meeting (zur Besprechung und Dokumentation) - gegebenfalls gibt es auch ein Sprint-Meeting mit unserem Betreuer."

Momentaner Status:
"Die Spiellogik ermöglicht eine individuelle Gestaltung - so ist unser Projekt derzeit fertig und spielbar, kann aber dennoch weiterentwickelt werden mithilfe von zusätzlichen Komponenten."
